## 背景

私は以前からドコモ回線の格安プランであるahamoを契約し、東京・神奈川エリアで日常的に使用していました。自宅や職場では問題なく通信できていたのですが、**ランチタイムや通勤時間帯の駅周辺、飲食店でのQRコード注文**など、人が集中するシーンで極端に通信が遅くなる、あるいはまったく繋がらない現象に悩まされていました。

「同じドコモ回線を使っているはずなのに、なぜこんなに遅いのか」——この疑問をきっかけに、原因を徹底的に調査することにしました。

## 発生した問題

具体的には以下のような症状が発生していました。

- 新宿・渋谷・横浜駅のホームでSNSが読み込めない（特に18時台）
- ランチ時の飲食店でQRコードメニューが開かず、店員を呼ぶ羽目になる
- 電車内で動画が止まり続ける（再生開始から30秒程度でバッファリング）
- 混雑時はスピードテストで **0.1〜1Mbps** しか出ない（本来なら20〜50Mbps出るエリア）

一見すると「単なる混雑」ですが、周囲のドコモ通常契約ユーザーは快適に使えているのを見て、**プランによる差別化（帯域制御）** が疑われました。

## 原因

調査の結果、ahamoが繋がらなくなる原因は主に4つあることが分かりました。

### 帯域制御の優先順位（最大の原因）

ahamoはNTTドコモのネットワークをそのまま使っていますが、**通信の優先順位が明確に異なります**。

| 優先度 | 対象プラン |
|--------|-----------|
| 高 | ドコモ ギガホ / ギガライト（通常契約） |
| 中 | **ahamo** |
| 低 | MVNO（OCN モバイル ONE / IIJmio など） |

ahamoはMVNOとほぼ同等の優先度で扱われています。つまり「同じドコモ回線でも、ahamoは輻輳時に帯域を絞られる設計」なのです。ドコモの通常契約ユーザーが優先的に帯域を確保し、ahamoユーザーはその残りを使う——という構造になっています。

### 人口密集エリアでの帯域逼迫

神奈川・東京は日本で最も人口密度の高いエリアです。特に以下のシチュエーションで顕著に影響が出ます。

- **飲食店のQRコード注文**：ランチ時（12〜13時）とディナータイム（18〜20時）は基地局が逼迫し、ahamoの低優先度ユーザーは弾き出される
- **電車内**：1両に数十〜百人以上が同時接続。基地局1基あたりの収容人数を超えると、優先度の低いahamoから切断される
- **主要駅**：新宿・渋谷・横浜・東京駅などの乗り換え駅は**数十万人が同時使用**するため、帯域がまったく足りない

### ahamo契約者の急増

ahamoは2021年3月にサービス開始。2024年9月時点で契約者数は**600万人超**に達しています。ユーザーが増えれば増えるほど、同じ帯域の奪い合いが激化するのは当然の結果です。ドコモとしてもahamoユーザーに帯域を割くと通常契約からの離脱を招くため、優先度を上げづらいという事情もあるでしょう。

### 5G/4Gハンドオーバーの問題（iPhoneユーザー向け）

iPhoneでahamoを使っている場合、**5Gと4Gの間のハンドオーバーがスムーズでない**ケースがあり、これが不安定さに拍車をかけています。5Gの電波は減衰しやすく、屋内や地下では4Gに切り替わりますが、この切り替え時に通信が一瞬途切れる——あるいは切り替わった先の4G帯域が逼迫しているため、結果として「繋がらない」状態になるケースがあります。

## 解決策

原因を特定した上で、いくつかの対策を検証しました。**すぐに試せる無料の対策**と**中長期的な有料の対策**に分けて紹介します。

### すぐに試せる対策（無料）

#### 4G（LTE）に固定する

iPhoneの場合、5Gをオフにして4G固定にするだけで安定するケースが多いです。ハンドオーバー問題を根本的になくせるので、まず試す価値があります。

```
設定 → モバイル通信 → 音声通話とデータ → 4G
```

Androidの場合は `設定 → ネットワークとインターネット → モバイルネットワーク → 優先ネットワークの種類 → LTE` で同様の設定が可能です。

#### キャリア設定をリセットする

キャリア設定のアップデート後に不具合が発生することがあります。リセットすることで改善する場合があります。

```
設定 → 一般 → 情報 → キャリア
```

「キャリア設定をリセット」があれば実行します。

#### ネットワーク設定をリセットする

より強力なリセット方法です。Wi-Fiのパスワードなども消える点に注意。

```
設定 → 一般 → 転送またはiPhoneをリセット → リセット → ネットワーク設定をリセット
```

#### 混雑時間帯を避ける

ahamoの低優先度の問題は「時間帯」に大きく依存します。以下の時間帯は特に回避推奨です。

- **ランチタイム**：12:00〜13:00
- **通勤ラッシュ**：8:00〜9:00 / 18:00〜19:00

#### 店のWi-Fiを積極的に使う

飲食店や商業施設のフリーWi-Fiは、ahamoよりもはるかに安定しています。混雑時間帯にどうしても通信が必要なら、Wi-Fiに逃げるのが最も確実です。

### 中長期的な対策（有料）

無料の対策で改善しない場合、**キャリアそのものの変更**が抜本的な解決になります。以下の選択肢を実際に比較検討しました。

#### povo2.0（au回線）

- **月額**：2,480円（20GB）
- **回線品質**：au本家と同等の優先度
- **評価**：ahamoより通信品質はマシというユーザー評価が多い。東京・神奈川の人口密集エリアでも比較的安定。
- **注意点**：au回線のエリア確認は必須。ドコモ回線よりややエリアが狭い場合がある。

#### LINEMO（SoftBank回線）

- **月額**：2,970円（30GB、ベストプランV）※旧スマホプラン（20GB/2,480円）は新規受付終了
- **回線品質**：比較的評価が高い
- **評価**：SoftBank回線は東京・神奈川の人口密集エリアで強いという声が多い。3大キャリアの中で最もahamoからの乗り換え先として評価が良い。
- **注意点**：データ超過後の速度制限（1Mbps）がやや厳しい。

#### 楽天モバイル（自社回線）

- **月額**：3,168円（Rakuten最強プラン、20GB超過後ギガ無制限）※2026年6月時点
- **回線品質**：自社回線エリアなら優先度高
- **評価**：料金面では最強だが、**エリアの問題**が大きい。東京23区や横浜市内でも自社回線が繋がらない場所が存在する。パートナー回線（au）エリアではMVNO相当の優先度に下がるため注意。
- **注意点**：楽天回線エリアマップを事前に必ず確認すること。

#### ドコモ通常契約に戻す

最後の選択肢ですが、最も確実な方法です。2026年6月時点で新規契約可能なドコモの通常プランは以下の通りです。

- **ドコモ MAX**（無制限）：基本料金8,448円/月、各種割引適用後5,148円/月〜
- **ドコモ ポイ活 MAX**（ギガ無制限＋dポイント還元）：6,578円/月〜
- **ドコモ mini**（4GB／10GB）：2,728円〜3,828円/月

これらのプランなら**ドコモ回線で最高の通信優先度**を享受できます。完全な使い放題を求めるなら「ドコモ MAX（無制限）」一択です。

- **注意点**：料金がahamoの1.7〜2.8倍になる（割引適用後でも5,000円超）。ただし「繋がらなさすぎてストレス」を考慮すれば、割高でも納得できる価値はある。

## 学び・補足

今回の調査で得られた最大の学びは「**同じ回線でもプランによって通信品質は大きく変わる**」という点です。

キャリアの格安プラン（ahamo/povo/LINEMO）は、あくまで「余剰帯域を安く譲ってもらう」というビジネスモデルです。混雑していない時間帯やエリアでは全く問題なく使える一方、**人が集中する場所・時間帯では低優先度の影響が如実に出ます**。

私自身は最終的に **povo2.0に乗り換え** ました。ahamoと比較すると、東京・神奈川の人口密集エリアでも通信が安定する場面が明らかに増えました（あくまで体感ですが）。とはいえ、ゴールデンウィークや年末年始の繁華街ではどの格安プランでも厳しい場面があることは理解しておいた方が良いでしょう。
